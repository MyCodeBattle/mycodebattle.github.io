# 博客文字大小优化计划

## 问题分析

根据对当前CSS文件的分析，我发现博客的文字大小设置存在以下问题：

1. **标题层级不清晰**：所有H1-H4标签在桌面端都使用相同的30px字体大小，缺乏视觉层次感
2. **移动端标题变化突兀**：从桌面端30px突然变为移动端1.25em，变化过大
3. **主题色不一致**：移动端H3-H4使用固定的blue颜色，与主题色#4A90E2不一致
4. **代码块字体大小可优化**：当前与正文相同，可适当调整以节省空间并提高可读性

## 优化建议

### 1. 建立清晰的标题层级

| 标题级别 | 桌面端大小 | 移动端大小 | 行高 | 间距 |
|---------|-----------|-----------|------|------|
| H1      | 2.25em (36px) | 1.75em (28px) | 1.3 | 1.5em 0 0.75em |
| H2      | 1.875em (30px) | 1.5em (24px) | 1.3 | 1.25em 0 0.625em |
| H3      | 1.5em (24px) | 1.25em (20px) | 1.4 | 1em 0 0.5em |
| H4      | 1.25em (20px) | 1.125em (18px) | 1.4 | 0.875em 0 0.4375em |

### 2. 统一主题色

- 确保移动端所有标题颜色与主题色#4A90E2一致
- 移除small.css中H3-H4的固定blue颜色设置

### 3. 优化代码块

- 将代码块字体大小调整为0.875em (14px)，提高阅读体验并节省空间
- 保持代码行高140%，确保可读性

### 4. 优化辅助文字

- 保持日期和分页器0.75em (12px)不变，这是合适的辅助文字大小
- 导航链接保持1em不变，确保清晰可见

## 实施步骤

1. 修改`css/default.css`文件：
   - 为H1-H4设置不同的字体大小
   - 调整标题的margin和padding
   - 优化代码块字体大小

2. 修改`css/small.css`文件：
   - 调整移动端标题大小，使其与桌面端平滑过渡
   - 移除H3-H4的固定blue颜色
   - 确保移动端标题层级清晰

3. 测试响应式效果：
   - 确保在不同屏幕尺寸下文字大小变化平滑
   - 验证所有标题级别在移动端和桌面端都有明显区分
   - 检查主题色一致性

通过以上优化，博客的文字大小将更加符合专业设计理念，具有清晰的层级结构、良好的可读性和一致的视觉风格。
